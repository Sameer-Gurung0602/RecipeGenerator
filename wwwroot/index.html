GET /api/recipes

Description:

Retrieves a list of all recipes in the Recipe Generator database. Each recipe includes details such as name, description, cook time, difficulty, ingredients, dietary restrictions, and favorite status for the user.

Query Parameters:

  • userId (optional, integer)
  • sortBy (optional, string)
  • sortOrder (optional, string)

Response Properties:

  • recipeId
  • name
  • description
  • cookTime
  • difficulty
  • createdAt
  • img
  • instructions
  • ingredients
  • dietaryRestrictions
  • isFavourite

Example Response:

Status: 200 OK

[
  {
    "recipeId": 23,
    "name": "Honey Ginger Shrimp",
    "description": "Sweet and spicy shrimp with ginger",
    "cookTime": 20,
    "difficulty": "Easy",
    "createdAt": "2025-01-23T00:00:00.000Z",
    "img": "https://images.unsplash.com/photo-1565557623262-b51c2513a641?w=400",
    "instructions": {
      "instructionsId": 23,
      "instruction": "1. Marinate shrimp. 2. Cook with ginger. 3. Add honey glaze."
    },
    "ingredients": ["Shrimp", "Ginger", "Honey"],
    "dietaryRestrictions": ["Gluten-Free", "Dairy-Free"],
    "isFavourite": false
  }
]

Expected Error Codes:

  • 404 Not Found – Invalid endpoint path
  • 500 Internal Server Error – Unexpected server error

GET /api/recipes/trending

Description:

Retrieves the most popular recipes based on fetch count (number of times each recipe has been viewed). Returns the top recipes sorted by popularity, perfect for displaying trending recipes on the homepage. Note: Viewing a recipe (GET /api/recipes/:id) automatically increments its fetch count.

Query Parameters:

  • count (optional, integer, default: 4, max: 10) - Number of trending recipes to return
  • userId (optional, integer, default: 1) - Used to determine if each recipe is in this user's favorites

Response Properties:

  • recipeId - Unique identifier for the recipe
  • name - Recipe name
  • description - Brief description of the recipe
  • cookTime - Time in minutes
  • difficulty - Difficulty level (Easy, Medium, Hard)
  • createdAt - ISO 8601 timestamp when the recipe was created
  • img - Image URL (HTTPS)
  • fetchCount - Number of times the recipe has been viewed
  • ingredients - Array of ingredient names
  • dietaryRestrictions - Array of dietary restriction names
  • isFavourite - Boolean indicating if the recipe is in the user's favorites

Example Request:

GET /api/recipes/trending (returns 4 recipes by default for user 1)

GET /api/recipes/trending?count=6 (returns up to 6 recipes for user 1)

GET /api/recipes/trending?count=6&userId=5 (returns up to 6 recipes with favorite status for user 5)

GET /api/recipes/:recipe_id

Parametric Endpoint:

:recipe_id - Integer representing the recipe's unique identifier

Description:

Retrieves a single recipe by its ID. Returns complete recipe information including title, description, instructions, ingredients, dietary restrictions, and favorite status for the user. Note: This endpoint automatically increments the recipe's fetch count (view counter) each time it is called, which affects the trending recipes ranking.

Query Parameters:

  • userId (optional, integer, default: 1) - Used to determine if the recipe is in this user's favorites

Example Request:

GET /api/recipes/23?userId=1

Example Response:

Status: 200 OK

{
  "recipeId": 23,
  "name": "Honey Ginger Shrimp",
  "description": "Sweet and spicy shrimp with ginger",
  "cookTime": 20,
  "difficulty": "Easy",
  "createdAt": "2025-01-23T00:00:00.000Z",
  "img": "https://images.unsplash.com/photo-1565557623262-b51c2513a641?w=400",
  "instructions": {
    "instructionsId": 23,
    "instruction": "1. Marinate shrimp. 2. Cook with ginger. 3. Add honey glaze."
  },
  "ingredients": ["Shrimp", "Ginger", "Honey"],
  "dietaryRestrictions": ["Gluten-Free", "Dairy-Free"],
  "isFavourite": false
}

Behavior Notes:

  • Fetch Count Tracking: Each call to this endpoint increments the recipe's view count by 1
  • Image URLs: All recipes include an HTTPS image URL
  • isFavourite: Set to true if the recipe is in the specified user's favorites, false otherwise
  • Instructions: May be null if no instructions exist for the recipe

Expected Error Codes:

  • 400 Bad Request – Invalid recipe_id format (must be a number)
  • 404 Not Found – Recipe with specified ID does not exist
  • 500 Internal Server Error – Unexpected server error

Parametric Endpoint:

:recipe_id - Integer representing the recipe's unique identifier

Description:

Retrieves all dietary restrictions associated with a specific recipe. Returns an array of dietary restriction names (e.g., Vegan, Gluten-Free).

Query Parameters:

None

Example Request:

GET /api/recipes/23/dietary-restrictions

Example Response:

Status: 200 OK

{
  "dietaryRestrictions": ["Gluten-Free", "Dairy-Free"]
}

Expected Error Codes:

  • 400 Bad Request – Invalid recipe_id format (must be a number)

  • 500 Internal Server Error – Unexpected server error

:userId - Integer representing the user's unique identifier

Description:

Retrieves all recipes that a specific user has saved to their favorites. Returns complete recipe information for each favorited recipe, including instructions, ingredients, dietary restrictions, and image URLs. All recipes in the response will have isFavourite set to true.

None

Response Properties (per recipe):

  • img - HTTPS image URL
  • instructions - Object containing instruction details
  • ingredients - Array of ingredient names (strings)
  • dietaryRestrictions - Array of dietary restriction names (strings)
  • isFavourite - Always true for this endpoint

    POST /api/favourites/:userId/recipes

    Parametric Endpoint:

    :userId - Integer representing the user's unique identifier

    Description:

    Adds a recipe to a user's favorites collection. The recipe must exist and must not already be in the user's favorites.

    Request Body Format:

    {
  "recipeId": integer (required, must be greater than 0)
}

    Request Body Validation:

    • recipeId is required and must be a positive integer (> 0)
    • Negative or zero values will return 400 Bad Request
    • Null or missing recipeId will return 400 Bad Request

    Example Request:

    POST /api/favourites/1/recipes

    {
  "recipeId": 23
}

    Example Response (Success):

    Status: 200 OK

    "Recipe saved to favourites successfully."

    Example Response (Already Favorited):

    Status: 404 Not Found

    "Recipe or User Not Found or recipe is already favourited"

    Expected Error Codes:

    • 400 Bad Request – Invalid request data (recipeId is null, zero, negative, or missing)
      Example: { "recipeId": 0 } or { "recipeId": -1 }
    • 404 Not Found – Recipe does not exist, user does not exist, or recipe is already favourited
      Note: The error message will contain "already favourited" if the recipe is already in the user's favorites
    • 500 Internal Server Error – Unexpected server error

    Concurrency Behavior:

    If multiple requests attempt to favorite the same recipe concurrently, only one will succeed. The database constraint ensures each recipe can only appear once per user's favorites. Subsequent requests will receive a 404 status.

    DELETE /api/favourites/:userId/recipes/:recipeId

    Parametric Endpoint:

    :userId - Integer representing the user's unique identifier
    :recipeId - Integer representing the recipe's unique identifier

    Description:

    Removes a recipe from a user's favorites collection. The recipe must exist and must currently be in the user's favorites for the operation to succeed.

    Query Parameters:

    None

    Request Body:

    None (parameters are in the URL path)

    Example Request:

    DELETE /api/favourites/1/recipes/23

    Example Response (Success):

    Status: 200 OK

    {
  "message": "Recipe removed from favourites successfully"
}

    Example Response (Not Found):

    Status: 404 Not Found

    {
  "message": "Recipe with ID 23 not found or not in user's favourites."
}
